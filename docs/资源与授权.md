# 资源与授权

本项目使用 Minecraft 风格资源和 TaC 武器资源做本地原型验证。资源链路要保持最小化、可追踪、可替换。

## 资源放置约定

- `public/assets/minecraft/`：浏览器运行时使用的方块、实体、物品、GUI 等 Minecraft 风格资源。
- `public/assets/fonts/`：Minecraft 风格字体。
- `public/assets/tacz/`：从 `tac-mod-resourcesV2` 精选复制的 TaCZ/Zero 新版武器资源。后续新增或替换枪械模型、贴图、HUD 图标、音效、换弹动画数据时优先放这里。
- `public/assets/tac/weapons/`：TaC 武器 2D 图标。
- `public/assets/tac/sounds/`：TaC 武器开火、换弹、抽枪音效。
- `public/assets/tac/effects/`：枪口火焰、命中标记等效果贴图。
- `public/assets/tac/models/p90/`：P90 Blockbench JSON 模型文件。
- `public/assets/tac/textures/p90/`：P90 模型贴图。
- `public/assets/tac/textures/crosshair/`：像素准星贴图（16×16），含 dot/circle/dynamic_default/better_default/round 五种风格，每把武器配不同准星。
- `public/assets/tac/textures/gui/`：GUI 像素素材，含 reloadbar（换弹条）、ammoslots（弹药槽）、firemode_auto/firemode_semi（射击模式图标）、armor_backdrop/armor_filler（护甲条底/填充）。阶段 2 已复制，后续 GUI 像素化迭代使用。

## 使用原则

- 只复制当前项目实际使用的资源，不复制整个外部模组。
- 新增资源时，优先选择最小集合：图标、必要音效、必要贴图、必要模型文件。
- 查找外部素材时按固定优先级处理：
  1. 优先查 `我的世界素材/tac-mod-resourcesV2/`，尤其是 `assets/tacz/custom/tacz_default_gun/` 下的新版枪械模型、贴图、音效、动画和参数。
  2. V2 没有合适素材时，再查 `我的世界素材/tac-mod-resources/` 下的旧版 TaC 资源。
  3. 两个 TaC 资源包都不合适时，最后查 `我的世界素材/block/`、`entity/`、`item/`、`gui/` 等原版 Minecraft 风格素材。
- 从 V2 复制的新资源默认落到 `public/assets/tacz/`；从旧 TaC 复制的资源继续落到 `public/assets/tac/`；原版素材落到 `public/assets/minecraft/`。
- 资源路径统一登记在 `src/config.js`，不要在业务代码中散落硬编码路径。
- 音频和模型资源要有失败回退，资源缺失不能导致游戏白屏或射击中断。

## 授权注意事项

- 当前资源用于本地原型和功能验证。
- 公开发布、上传商店、课程分发或商业使用前，需要单独确认 Minecraft 风格资源和 TaC 资源的授权边界。
- 如果资源来自 GPL 或其他传染性许可证项目，要先确认是否会影响本项目整体发布方式。
- 后续替换为自制资源或明确可商用资源时，需要同步更新本文档。

## 资源验收

- 新增资源后运行资源存在性测试，确认路径和文件名一致。
- 浏览器 Network 中不应出现大量 404、重复请求或长时间 pending 的资源。
- 音效文件应避免在持续射击时重复 fetch；模型失败时应有明确 fallback。
