# 武器系统

武器系统目前支持 Glock 17、M4、AK47、AWP 和 P90。5 把武器均使用第一人称 3D 模型；第一人称 2D overlay、2D fallback 和 `debugWeapon2D` 已移除。PNG 武器图标资源仍保留在素材目录中，但不再参与第一人称战斗渲染。

5 把武器统一走 `createWeaponModel` 通用路径：读取 Blockbench JSON，运行时由 `buildFirstPersonBlockbenchMesh` 生成第一人称实体模型。材质按 weaponId 分策略——P90 按 texture key 选 `P90_MATERIAL_COLORS` 多色（`#2`/`#3`/fallback），其他 4 把按 element 尺寸选 base/accent 双色。位置、旋转、缩放和枪口锚点均由 `modelConfig` 管理。

## 配置字段

武器配置集中在 `src/config.js` 的 `WEAPON_CONFIG`，武器顺序由 `WEAPON_ORDER` 控制。

- `id`：武器内部标识，需要和 `WEAPON_CONFIG` 的键一致。
- `label`：HUD 显示名称。
- `slot`：热栏槽位，当前对应数字键 `1` 到 `5`。
- `magazineSize`：弹匣容量。
- `reloadDuration`：换弹耗时，单位为秒。
- `fireInterval`：两次射击之间的最短间隔，数值越小射速越快。
- `automatic`：是否按住鼠标自动连发。
- `bodyDamage`：身体命中的基础伤害。
- `recoil`：第一人称武器后坐力表现强度。
- `cameraKick`：开火时相机上跳幅度。
- `fireSound`、`reloadSound`、`drawSound`：音效 key，对应 `SOUND_PATHS`。
- `iconPath`：武器 PNG 图标资源路径，目前保留为素材引用，不作为第一人称 fallback。
- `modelPath`：3D 模型路径，5 把武器统一指向 Blockbench JSON（`ASSET_PATHS.weaponModels[weaponId]`）。
- `modelConfig`：3D 第一人称模型参数，包含 `position`、`rotation`、`scaling` 和 `muzzleLocalPosition`。
- `muzzleFlash`：枪口火焰视觉参数，包含 `size`（世界单位大小）、`alpha`（透明度 0-1）、`rotationRandom`（每次开火贴图随机旋转弧度）。未配置时使用默认值 size=0.3 / alpha=0.85。
- `tracerInterval`：每隔多少发显示一次弹道。
- `crosshair`：像素准星配置。`image` 指向 `ASSET_PATHS.crosshair` 中的贴图路径；`hiddenByAds` 为 true 时表示开镜后隐藏普通准星（仅 AWP 使用）。
- `ads`：瞄准镜配置（仅 AWP）。`fov` 为开镜后视野弧度（0.35≈20°），`sensitivityScale` 为鼠标灵敏度倍率（0.5=开镜后灵敏度减半）。

## 像素准星与 AWP 开镜

阶段 2 将准星从 3 个 Rectangle 重构为 1 个 `GUI.Image`，每把武器配不同风格的 16×16 像素贴图，武器切换时调用 `setCrosshairForWeapon(ui, weaponId)` 切换 `source`。命中染色仍走 `setCrosshair(ui, state, enabled)`，通过 `control.color` 给 Image 着色。

当前准星分配：

| 武器 | 准星贴图 | 风格说明 |
|------|----------|----------|
| Glock 17 | `crosshair.dot` | 单点，手枪精准瞄准 |
| M4 | `crosshair.round` | 圆形+中心十字，步枪清晰瞄准 |
| AK47 | `crosshair.better` | 精细十字，高后坐力武器配清晰准星 |
| AWP | `crosshair.dot` | 开镜时隐藏，显示瞄准镜蒙版 |
| P90 | `crosshair.circle` | 圆圈，近距离快速瞄准 |

AWP 开镜逻辑在 `src/main.js`：右键（`pointerdown` button=2）切换 `state.ads`，每帧用 `Scalar.Lerp` 平滑过渡 `camera.fov` 到 `weapon.ads.fov`（0.35），同时降低 `camera.angularSensibility`（除以 `sensitivityScale`）。开镜时显示 `ui.scopeOverlay`（瞄准镜蒙版）并隐藏普通准星，关镜时反向恢复。为避免 Babylon 相机输入抢占右键事件，已将 `camera.inputs.attached.pointers.buttons` 设为 `[0]`（仅左键），右键完全由 `pointerdown` handler 接管。指针锁定退出时自动重置 ads/FOV/灵敏度，防止状态残留。

瞄准镜蒙版（`buildScopeOverlay`）由三层 GUI 组成：外圈半透明黑色遮罩（`rgba(0,0,0,0.55)`）+ 中心 `circle.png` 放大到 500px 作镜框 + 红色十字线（横竖各 1 条，alpha 0.5）。切换武器时自动重置 ads/FOV/灵敏度。

## 新增武器流程

1. 在 `public/assets/tac/weapons/` 放入武器图标素材，作为热栏或后续 UI 资源来源。
2. 如有音效，在 `public/assets/tac/sounds/` 放入最小必要文件，并在 `SOUND_PATHS` 中登记。
3. 在 `public/assets/tac/models/` 放入 Blockbench JSON 或其他 3D 模型资源，并在 `ASSET_PATHS.weaponModels` 登记。
4. 在 `WEAPON_CONFIG` 新增武器配置，并把 id 加入 `WEAPON_ORDER`。
5. 必须提供 `modelConfig.position/rotation/scaling/muzzleLocalPosition`；不再新增 `display` 字段。
6. 必须提供 `crosshair.image`，从 `ASSET_PATHS.crosshair` 选一张 16×16 像素贴图；狙击类武器额外配 `ads` 和 `crosshair.hiddenByAds`。
7. 为弹匣、射速、换弹、自动/半自动、身体伤害、音效、模型结构、枪口锚点和准星配置补充测试。
8. 浏览器中验证模型可见、开火、换弹、枪口火焰锚点、准星切换、命中反馈、控制台错误和网络请求数量。

## 3D 武器模型

第一人称 3D 模型由 `src/weaponModel.js` 管理，运行时由 `src/main.js` 根据当前武器选择对应 controller。

- 选择任意 3D 模型 ready 的武器时，显示对应模型；模型未 ready 或加载失败时不显示第一人称武器，测试应失败并暴露问题。
- Glock 17、M4、AK47、AWP 的运行时第一人称模型先使用纯色材质；标准 Blockbench face 还原函数仍保留，用于结构验证和后续贴图化迭代。
- P90 的 Blockbench JSON、贴图路径和多色材质映射（`P90_MATERIAL_COLORS`）需要保留测试或浏览器验收。
- 调整模型位置时，要同时检查不遮挡准星、不会穿过屏幕边缘、换弹/后坐力表现可接受。
- 调整枪口火焰时不要再写相机坐标 `muzzleOffset`，应校准 `modelConfig.muzzleLocalPosition`。

### 枪口火焰锚点

每个 weapon controller 都有 `muzzleAnchor`，挂在对应武器 root 下。`muzzleFlashPlane` 不再 parent 到 camera，而是每帧复制当前武器 `muzzleAnchor.getAbsolutePosition()`，并用 billboard 面向相机。

枪火 plane 使用 `BILLBOARDMODE_ALL` 始终朝向相机，因此 mesh rotation 会被每帧覆盖；旋转表现改用 `material.diffuseTexture.wAng` 在每次开火时随机化。大小通过 plane scaling 控制（基础 plane 为 1×1，`scaling` 即世界单位大小），透明度通过 `material.alpha` 控制，两者都按当前武器的 `muzzleFlash` 配置每帧应用。

验收重点：

- 枪口火焰屏幕坐标应贴近 `muzzleAnchorScreen`。
- `muzzleAnchorScreen` 应位于武器屏幕投影的枪管前端区域，不能落在枪身中部或枪托附近。
- 开火、后坐力和换弹时，枪口火焰应跟随武器 root 一起移动。

### E2E 调试探针

`/?e2e=1` 会暴露 `window.__blockTargetRangeDebug`，用于 Playwright 校准和验收：

- `start()`：进入游戏流程。
- `selectWeapon(weaponId)`：切换到指定武器。
- `shoot()`、`reload()`：触发开火和换弹。
- `setModelConfig(weaponId, modelConfig)`：运行时临时覆盖某把武器的 3D 校准参数，支持调整 `muzzleLocalPosition`。
- `snapshot()`：返回当前武器、模型加载状态、可见 mesh 数、`modelConfig`、屏幕投影范围、枪口锚点和枪口火焰屏幕坐标。

验收时不能只看 `visibleMeshCount`。mesh 存在不代表玩家屏幕里看得见；必须同时检查 3D root enabled、无 2D weapon plane、屏幕投影面积、位置是否在右下区域，以及枪口锚点是否落在合理枪管前端。

### P90 材质颜色

- `P90_MATERIAL_COLORS`（`src/weaponModel.js`）刻意走中性深枪灰/黑灰金属色（R≈G≈B 略偏冷），避免蓝色天光叠加后显青。
- Blockbench 路径用 `colorMaterial` 纯色材质，纹理键 `#2`/`#3` 映射到对应色值。
- 未命中 `#2`/`#3` 的 element 回退到 `P90_MATERIAL_COLORS.fallback`，保证所有 mesh 最终都呈黑灰金属色。
- 修改色值需同步收紧 `test/weaponModel.test.js` 的非青阈值（`b < r + 12`、`g < r + 12`）。
