# 武器系统

武器系统目前支持 Glock 17、M4、AK47、AWP 和 P90。前四把主要使用 2D 图标作为第一人称显示，P90 额外尝试加载 3D 第一人称模型，失败时回退到 2D 图标。

## 配置字段

武器配置集中在 `src/config.js` 的 `WEAPON_CONFIG`，武器顺序由 `WEAPON_ORDER` 控制。

- `id`：武器内部标识，需要和 `WEAPON_CONFIG` 的键一致。
- `label`：HUD 显示名称。
- `slot`：热栏槽位，当前对应数字键 `1` 到 `5`。
- `magazineSize`：弹匣容量。
- `reloadDuration`：换弹耗时，单位为秒。
- `fireInterval`：两次射击之间的最短间隔，数值越小射速越快。
- `automatic`：是否按住鼠标自动连发。
- `bodyDamage`：身体命中的基础伤害。
- `recoil`：第一人称武器后坐力表现强度。
- `cameraKick`：开火时相机上跳幅度。
- `fireSound`、`reloadSound`、`drawSound`：音效 key，对应 `SOUND_PATHS`。
- `iconPath`：2D 武器图标路径。
- `modelPath`：3D 模型路径，目前主要用于 P90。
- `tracerInterval`：每隔多少发显示一次弹道。
- `display`：2D 第一人称显示参数，包含位置、缩放、旋转和镜像。

## 新增武器流程

1. 在 `public/assets/tac/weapons/` 放入武器图标。
2. 如有音效，在 `public/assets/tac/sounds/` 放入最小必要文件，并在 `SOUND_PATHS` 中登记。
3. 在 `WEAPON_CONFIG` 新增武器配置，并把 id 加入 `WEAPON_ORDER`。
4. 如新增槽位，同步检查键盘切枪范围、HUD 热栏、测试用例和浏览器验收。
5. 为弹匣、射速、换弹、自动/半自动、身体伤害、音效和资源存在性补充测试。
6. 浏览器中验证开火、换弹、切枪、命中反馈、控制台错误和网络请求数量。

## 2D 武器显示校准

2D 武器图标不一定天然适合第一人称视角，需要逐把校准。

- 枪口必须朝向敌人方向，枪把应落在玩家右下角附近。
- 优先用 `?debugWeapon2D=1` 或专门调试视图同屏检查所有武器。
- 校准顺序建议为：先修镜像方向，再微调 `rotationZ`，最后调 `offsetX`、`offsetY` 和 `scale`。
- 镜像实现要保持稳定，避免因为材质背面、平面旋转或透明贴图导致不同武器表现不一致。

### 当前校准结论

- glock17、m4、ak47、awp：背包图标不是第一人称 sprite，当前以 `flipX: true`、`flipY: false` 的水平镜像作为基准；`flipY` 会破坏枪身上下关系并把枪口压向热栏。
- p90：背包图标 native 枪口已在左上，不翻转（`flipX: false`、`flipY: false`）。
- 校准依据：以用户提供的 `debugWeapon2D` 验收截图和浏览器实测为最高优先级，像素分析只能辅助，不能替代真实画面。
- 改动任一 display 值需重新校准并同步更新 `test/assets.test.js` 的快照断言。

## P90 3D 模型

P90 第一人称模型由 `src/weaponModel.js` 管理。

- 选择 P90 且 3D 模型 ready 时，隐藏 2D 武器平面并显示 3D 模型。
- 模型加载失败时必须回退 2D 图标，不能白屏或影响其他武器。
- glTF、Blockbench JSON、贴图路径和 fallback 行为需要保留测试或浏览器验收。
- 调整模型位置时，要同时检查不遮挡准星、不会穿过屏幕边缘、换弹/后坐力表现可接受。

### P90 材质颜色

- `P90_MATERIAL_COLORS`（`src/weaponModel.js`）刻意走中性深枪灰/黑灰金属色（R≈G≈B 略偏冷），避免蓝色天光叠加后显青。
- Blockbench 路径用 `colorMaterial` 纯色材质，纹理键 `#2`/`#3` 映射到对应色值。
- glTF fallback 路径覆盖所有可见 mesh 的材质为 `P90_MATERIAL_COLORS.fallback`，保证两条路径最终都呈黑灰金属色。
- 修改色值需同步收紧 `test/weaponModel.test.js` 的非青阈值（`b < r + 12`、`g < r + 12`）。
