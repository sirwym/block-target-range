# 项目结构

本文说明当前项目的主要目录和运行链路，方便后续修 bug 或扩展玩法时快速定位。

## 目录职责

- `src/`：游戏源码，包含 Babylon.js 场景、玩家输入、敌人、武器、音频、UI、特效和资源加载。
- `test/`：Node test 测试，覆盖武器规则、音频节流、资源存在性、碰撞、评分、TaCZ 模型数据等稳定逻辑。
- `public/assets/`：浏览器运行时直接访问的资源，包含 Minecraft 风格贴图、字体、TaC/TaCZ 武器图标、音效、5 把武器的 Blockbench JSON 模型和贴图。后续从 `tac-mod-resourcesV2` 精选的新资源放入 `public/assets/tacz/`，旧 TaC 资源继续放入 `public/assets/tac/`。
- `assets/`：项目保留的源资源目录，当前主要用于记录或对照 Minecraft 风格资源。
- `dist/`：构建输出目录，由 `npm run build` 生成，不作为源码维护入口。

## 核心模块

- `src/main.js`：游戏入口和主循环，连接输入、场景、射击、命中、HUD、敌人生成和调试参数。
- `src/config.js`：集中配置游戏参数、资源路径、音效路径、武器列表和武器属性。
- `src/weapon.js`：纯武器状态逻辑，负责弹匣、射击冷却、换弹、切枪和自动/半自动限制。
- `src/audio.js`：音频上下文、真实 OGG 音效播放、fallback 合成音和节流缓存。
- `src/weaponModel.js`：5 把武器第一人称 3D 模型统一加载（Blockbench JSON → CreateBox 实体模型）和显示更新。
- `src/effects.js`：命中特效、弹道、方块碎片、飘字和临时 mesh 生命周期。
- `src/ui.js`：Babylon GUI HUD、准星、热栏、结算面板和调试标签。
- `src/actors.js`：敌人生成、动画、移动和血条结构。
- `src/world.js`：地面、跑道、基地和灯光等场景结构。
- `src/assets.js`：纹理、材质、皮肤贴图和通用材质工具。

## 主流程

1. `main.js` 初始化 Babylon 引擎、相机、场景、资源、世界和 UI。
2. 玩家点击开始后进入倒计时，再进入正式 `playing` 状态。
3. 键盘控制移动、跳跃、换弹和 `Digit1` 到 `Digit5` 切枪；鼠标按下触发射击或自动武器连发。
4. 射击先走 `weapon.js` 扣弹和冷却，再播放音效、后坐力、枪口火焰和弹道。
5. 准星射线命中敌人时进入命中结算，更新血量、飘字、得分、连击、击杀和 HUD。
6. 每帧更新敌人移动、玩家状态、武器表现、3D 武器模型、临时特效和胜负条件。

## 维护提示

- 能放在 `weapon.js`、`combat.js`、`collision.js` 这类纯逻辑模块里的规则，优先不要写进 `main.js`。
- 涉及 Babylon.js 坐标、父子节点、贴图镜像、模型 fallback 的代码，要写必要中文注释说明坐标系和回退原因。
- 修改资源路径时，同时检查 `config.js`、`public/assets/` 和资源存在性测试。
