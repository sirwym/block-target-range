# 调试与验收

本文记录当前靶场项目的调试入口、浏览器验收清单和常见故障排查方向。

## 调试参数

在本地开发服务器 URL 后追加参数即可启用调试视图，例如 `/?debugWeapon=1`。

- `debugHitbox=1`：显示或辅助检查命中盒。
- `debugActor=1`：显示敌人结构、贴图或行为相关调试信息。
- `debugWeapon=1`：检查当前武器显示、P90 模型状态和第一人称武器表现。
- `e2e=1`：启用 Playwright 自动验收探针 `window.__blockTargetRangeDebug`，可启动游戏、切换武器、开火、换弹、临时写入 `modelConfig`，并读取 3D 模型屏幕投影和枪口锚点数据。
- `mode=weaponLab`：进入武器试验场模式。无敌人/倒计时/基地血量，只有地面 + 弹道墙 + 武器 + 数据看板。可配合 `e2e=1` 使用（`?e2e=1&mode=weaponLab`），`snapshot().weaponLab` 暴露弹孔数和双层统计。

## 浏览器验收清单

玩法、资源、音频、模型或 UI 改动后，建议至少完成以下检查：

- 页面正常进入游戏，控制台没有红色错误。
- `Digit1` 到 `Digit5` 可以切换 Glock 17、M4、AK47、AWP、P90。
- 5 把武器都显示 3D 模型，没有第一人称 2D weapon plane。
- 每把枪可以射击、扣弹、换弹，HUD 弹药显示正确。
- 自动武器按住鼠标能持续射击，不会几发后卡住。
- 枪口火焰位于枪管前端，开火、后坐力和换弹时跟随 3D 武器移动。
- 命中敌人后有命中反馈、血条、飘字、得分和连击反馈。
- 3D 武器验收不能只看 mesh 数量。必须确认模型在屏幕右下角有足够投影面积、枪形可辨、无 2D fallback；“mesh 已加载但主画面几乎看不到”视为未修复。
- Network 中 OGG、模型和贴图请求数量合理，没有几分钟几万条重复请求。
- 结束一局后结算面板、重新开始、音效和控制状态正常。

## 武器试验场验收清单

访问 `http://127.0.0.1:5174/?mode=weaponLab` 进入武器试验场模式，重点检查：

- 页面只有地面 + 弹道墙 + 武器 + 左上数据看板，无敌人/倒计时/基地血量。
- 原靶场 HUD（Score/Time/基地血量/经验条/tip）不显示，准星 + 热栏（武器图标+弹药+武器名）正常。
- 第一次点击 canvas 同时请求鼠标锁定 + 开火，后续点击直接开火；Esc 退出锁定后点击 canvas 重新锁定，不弹开始面板。
- `Digit1` 到 `Digit5` 可切换 5 把武器，枪火跟随枪口。
- 射击弹道墙时弹孔在命中点出现并累积保留；瞄准地面开火只扣弹不生成弹孔，命中率下降（地面命中算脱靶）。
- 长按自动武器（M4/AK47/P90）能持续射击，弹孔轨迹反映后坐力上跳。
- `T` 键清除所有弹孔。
- 换弹后弹匣层统计重置，会话层统计保留。
- 切武器后会话层统计重置，弹孔不受影响。
- 左上数据看板显示弹匣层 5 行（射击/命中/命中率/DPS/射速）和会话层 4 行（累计射击/累计命中/累计伤害/会话DPS）。
- 会话 DPS 停火后随时间下降（口径为从首枪到当前 tick），弹匣层 DPS 不含换弹间隙（口径为该弹匣首发到末发）。
- 控制台无错误，Network 无异常请求暴涨。
- 用 `?e2e=1&mode=weaponLab` 探针确认 `snapshot().mode === "weaponLab"`、`weaponLab.bulletHoleCount` 和 `weaponLab.stats` 数据结构正确。
- `npm run test:e2e` 包含 `e2e/weaponLab.spec.js` 19 个专属用例：无控制台错误、mode 正确、开火生成弹孔+统计累加、换弹重置弹匣层、切武器重置会话层、T 键清弹孔、截图、Tab 键锁定相机、准星切换、AWP 开镜、死靶放置/射击统计/3s 重生/H 清除、敌人模式启动/刷怪/射击统计/抵达扣血/HP 归零 defeat/H 清空。

### 阶段 1：角色移动验收清单

- WASD 能移动，Shift 能跑步，空格能跳跃（跳跃保留靶场手感）。
- 走到地面边缘（x≈±18 或 z≈18）被 `playerBounds` 软限制挡住，不掉空。
- 走到弹道墙前（z≈-2）不再前进（zMin 限制）。
- 按住 Tab 键：相机锁定（不能移动也不能转头），方便观察弹孔分布；松开 Tab 恢复。
- Tab 键不触发浏览器焦点切换（已 preventDefault）。
- 底部操作提示显示 weaponLab 专属文案："WASD 移动  Shift 跑步  空格跳跃  Tab 锁定相机  鼠标射击  R 换弹  1-5 切枪  T 清弹孔"。
- 5 把武器切换后枪口火焰锚点、3D 模型朝向仍正确（重跑 `e2e/weapons.spec.js` 验证）。
- 用 `?e2e=1&mode=weaponLab` 探针确认 `snapshot().weaponLab.cameraLocked` 和 `snapshot().weaponLab.playerPosition` 数据正确。
- 控制台无错误。

### 阶段 2：GUI 美化验收清单

访问 `http://127.0.0.1:5174/?mode=weaponLab` 进入武器试验场，重点检查准星和 AWP 开镜：

- 切换 5 把武器（`Digit1`-`Digit5`）时，准星贴图跟随切换：
  - Glock 17 → 单点（dot）
  - M4 → 圆形十字（round）
  - AK47 → 精细十字（better）
  - AWP → 单点（dot，开镜后隐藏）
  - P90 → 圆圈（circle）
- 命中弹道墙时命中标记（hit_marker）短暂显示后淡出（约 0.15s），不常亮。
- 切换到 AWP 后右键点击开镜：
  - 视野平滑缩小（FOV 从 72°→20°），有过渡感而非瞬切。
  - 鼠标灵敏度降低，开镜后转头变慢。
  - 瞄准镜蒙版显示：外圈半透明黑色遮罩 + 中心圆形镜框 + 红色十字线。
  - 普通准星在开镜时隐藏。

- 切换到其他武器后再切回 AWP，开镜状态已重置（不会残留开镜）。
- 用 `?e2e=1&mode=weaponLab` 探针确认 `snapshot().weaponLab.ads`、`snapshot().weaponLab.crosshair`、`snapshot().weaponLab.fov` 数据正确。
- `npm run test:e2e` 包含准星切换和 AWP 开镜 2 个 E2E 用例。

### 阶段 3：死靶模式验收清单

访问 `http://127.0.0.1:5174/?mode=weaponLab` 进入武器试验场，按 G 键放置死靶，重点检查：

- 按 G 键在准星命中地面的位置生成静止假人（zombie），不调用 `nextLaneIndex`。
- 死靶位置精确：x/z 等于命中点，y=0.04 贴地，不随 lane 偏移。
- 死靶被击中后不 z 后退（`updateTarget` 对 `isDummy` 跳过后退），避免累积前滑。

- 死靶血量归零后隐藏模型（不 dispose），3 秒后原地重生（恢复血量+重新显示+重置血条）。
- 死靶上限 8 个，超过后滚动清除最早的。
- 数据看板左下区块显示死靶统计：死靶数/爆头/身体/爆头率。
- 按 H 键清除所有死靶，mode 回到 idle。
- 敌人模式（B 键）启动后 G 键被禁用，防止误放死靶。

### 阶段 4：敌人模式验收清单

访问 `http://127.0.0.1:5174/?mode=weaponLab` 进入武器试验场，按 B 键启动 60s 生存挑战，重点检查：

- 按 B 键启动敌人模式：清空已有死靶/敌人，mode 切换为 "enemy"，计时器 60s、HP 5、首次刷怪延迟 0.5s。
- 敌人从 z=-18（墙后方）车道生成，沿 +z 方向推进玩家。
- 刷怪节奏跟随波次：0-25s warmup（间隔慢、速度慢）、25-55s mixed（间隔缩短、creeper 概率出现、速度提升）、55-60s rush（间隔最短、速度最快）。
- 同屏敌人上限 8 个，达到上限后不再刷怪，待有敌人被击杀或抵达后继续。
- 射击敌人：爆头累加 `enemyHeadshots`，身体命中累加 `enemyBodyshots`，击杀累加 `enemyKills`；敌人被击杀后直接 dispose 不重生。

- 胜利/阵亡后看板时间行显示"存活！"/"阵亡！"，不再倒计时。

## 测试优先级

测试不是最后补流程，而是本项目防止玩法回归的主要护栏。修改武器、音频、资源、命中、碰撞、模型或 UI 状态时，先判断是否需要补测试，再开始验收。

- 自动测试优先覆盖稳定规则：武器状态、弹匣和换弹、自动/半自动射击、音频缓存、资源存在性、碰撞、评分、模型结构。
- 浏览器验收优先覆盖真实体验：持续射击、鼠标按住连发、控制台错误、Network 请求数量、3D 武器可见性、枪口锚点和 HUD 状态。
- 只跑自动测试不能替代浏览器验收；枪模朝向、模型遮挡、枪口火焰位置、音频请求暴涨这类问题必须在真实页面里看。
- 运行 `npm run test:e2e` 会通过 Playwright/Chromium 进入 `?e2e=1` 验收模式，捕获页面加载期控制台错误，并验证 1-5 号武器进入靶场后使用 3D 模型。
- `npm run test:e2e` 会输出单武器截图、静态 contact sheet 和开火 contact sheet，用于人工复核 Glock 17、M4、AK47、AWP、P90 的屏幕位置和枪口火焰。
- 如果修复线上或手动复现过的问题，测试里要尽量留下对应的回归用例，避免以后重复踩坑。

## 常见故障排查

### 命中后卡住

优先检查控制台是否出现命中反馈相关错误。飘字、特效、HUD 更新不能中断射击主链路；即使视觉反馈失败，也要保证扣血、扣弹、计分和主循环继续运行。

### 音频请求暴涨

检查 `src/audio.js` 的缓存状态。每个 OGG 应该最多加载一次；加载失败后也要记录失败状态，后续播放 fallback，不能反复 fetch 同一路径。

### 枪口火焰位置错误

先看 `?e2e=1` 的 `snapshot().runtime.muzzleAnchorScreen`、`muzzleFlashScreen` 和 `muzzleFlashDistancePx`。如果火焰在枪身中部，优先调 `modelConfig.muzzleLocalPosition`；不要再添加相机坐标式 `muzzleOffset`。

调试枪火视觉（大小/透明度/旋转）时，通过 `snapshot().muzzleFlash` 读取当前应用配置。运行时可用 `setModelConfig` 临时改 `muzzleLocalPosition`，但 `muzzleFlash` 视觉参数需直接改 `src/config.js` 后刷新页面查看。枪火旋转通过贴图 `wAng` 实现，不再通过 mesh rotation（`BILLBOARDMODE_ALL` 会覆盖后者）。

### 3D 模型异常

先区分模型资源加载失败、Blockbench JSON 解析问题、贴图/材质问题和模型坐标问题。第一人称 2D fallback 已移除，模型异常应通过测试或控制台直接暴露。

如果 e2e 里 `visibleMeshCount` 很高但截图里武器不可辨，优先看 `snapshot().activeModel.screenBounds`：投影宽高、面积比例和中心点才是第一人称视觉是否修好的关键证据。

P90 已和其他 4 把武器统一走 `createWeaponModel` 通用路径（Blockbench JSON → `buildFirstPersonBlockbenchMesh`）；P90 在材质选择阶段按 texture key 选 `P90_MATERIAL_COLORS` 多色，其他武器按 element 尺寸选 base/accent 双色。排查任意武器模型异常时都看 `snapshot().activeModel`（source/partCount/screenBounds/visibleMeshCount）。

### 构建或测试失败

先运行 `npm run test` 定位逻辑测试，再运行 `npm run build` 定位构建或资源路径问题。完整提交前使用 `npm run check`，视觉相关改动还需要单独运行 `npm run test:e2e`。
